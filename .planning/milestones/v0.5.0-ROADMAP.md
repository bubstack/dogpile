# Milestone v0.5.0: Observability and Auditability

**Status:** ✅ SHIPPED 2026-05-02
**Phases:** 6–10
**Total Plans:** 23
**Requirements:** 13/13

## Overview

Full observability stack for Dogpile: per-event provenance metadata, typed event introspection, per-run health diagnostics, an independent versioned audit record, a duck-typed OTEL tracing bridge, and a named-counter metrics hook — all with zero new required dependencies and replay-safe contracts.

## Phases

**Goal:** Every model request and response event in a completed trace carries structured provenance metadata (model id, provider id, call id, ISO-8601 timestamps), and that metadata is JSON-serializable and replay-stable.
**Depends on:** Nothing (first phase of this milestone)
**Requirements:** PROV-01, PROV-02
**Plans:** 6 plans

- [x] 06-04-PLAN.md — /runtime/provenance subpath module + package.json wiring
- [x] 06-05-PLAN.md — Contract tests + frozen provenance-event-v1.json fixture
- [x] 06-06-PLAN.md — CHANGELOG v0.5.0 entry + CLAUDE.md invariant update

**Details:**

**Goal:** Callers can inject an optional duck-typed OTEL-compatible tracer on `EngineOptions`; when present the SDK emits spans for run start/end, sub-run start/end, and agent turn start/end with correct parent-child ancestry; when absent runs complete with zero span overhead.
**Depends on:** Phase 6
**Requirements:** OTEL-01, OTEL-02, OTEL-03
**Plans:** 5 plans

Plans:
- [x] 09-00-PLAN.md — Wave 0: createDelegatingDeterministicProvider in src/testing/
- [x] 09-01-PLAN.md — Tracing module surface (DogpileTracer/Span/Options + DOGPILE_SPAN_NAMES) + tracer? on EngineOptions/DogpileOptions + devDeps
- [x] 09-02-PLAN.md — Engine span lifecycle: runNonStreamingProtocol + streaming execute() + RunProtocolOptions.parentSpan + replay tracing-free comments
- [x] 09-03-PLAN.md — Public-surface lockstep: /runtime/tracing subpath + package-exports.test.ts + no-otel-imports.test.ts + otel-tracing-contract.test.ts
- [x] 09-04-PLAN.md — Docs lockstep: CHANGELOG + CLAUDE.md invariant + docs/developer-usage.md OTEL section + pnpm run verify

**Details:**
- `tracer?: DogpileTracer` on `EngineOptions` and `DogpileOptions`; duck-typed, no `@opentelemetry/*` import in SDK source.
- Span names: `dogpile.run`, `dogpile.sub-run`, `dogpile.agent-turn`, `dogpile.model-call`.
- Sub-run spans nest under parent run spans via `parentRunIds` ancestry chain.
- Caller bridges real OTEL tracer via a thin WeakMap bridge (documented in `docs/developer-usage.md`).
- `no-otel-imports.test.ts` enforces zero OTEL imports in runtime/browser/providers.
- `replay()` and `replayStream()` are intentionally tracing-free.

### Phase 10: Metrics / Counters

**Goal:** Callers can supply an optional metrics hook on `EngineOptions` and receive named counters (token usage, cost, turn count, duration) at run and sub-run completion; omitting the hook adds zero overhead.
**Depends on:** Phase 9
**Requirements:** METR-01, METR-02
**Plans:** 4 plans

Plans:
- [x] 10-01-PLAN.md — MetricsHook/RunMetricsSnapshot types + metrics.ts module + metricsHook?/logger? on EngineOptions/DogpileOptions
- [x] 10-02-PLAN.md — Engine integration: MetricsState + openRunMetrics + handleMetricsEvent + closeRunMetrics + fireHook + createEngine threading
- [x] 10-03-PLAN.md — Public-surface lockstep: /runtime/metrics subpath + package-exports.test.ts + metrics-contract.test.ts + frozen fixture
- [x] 10-04-PLAN.md — Docs lockstep: CHANGELOG + CLAUDE.md invariant + docs/developer-usage.md Metrics section + pnpm run verify

**Details:**
- `metricsHook?: MetricsHook` and `logger?: Logger` on `EngineOptions` and `DogpileOptions`.
- `RunMetricsSnapshot`: `inputTokens`, `outputTokens`, `costUsd`, `turns`, `durationMs` plus `runId`, `depth`, `protocol`, `outcome`.
- `onRunComplete` fires only at root depth (0); `onSubRunComplete` fires per child `sub-run-completed` event.
- Hook throws and async rejections routed to `logger.error` — never propagate into run result.
- `replay()` and `replayStream()` intentionally ignore `metricsHook`.
- `@dogpile/sdk/runtime/metrics` is a public subpath; `MetricsHook` and `RunMetricsSnapshot` are subpath-only (no root re-export).

## Milestone Summary

**Key Decisions:**
- OTEL bridge uses duck-typed tracer injection with a caller-side WeakMap bridge; no `@opentelemetry/*` in runtime source. Structural identity with real OTEL `Tracer` was not achievable given parent-context differences.
- `RunResult.health` is required (not optional) — all construction paths (run, replay, stream, delegate child) compute health before returning.
- Audit record is type-independent from `RunEvent` variants; callers can reference `AuditRecord` without importing event types.
- Phase 9 uses internal `runProtocol` wrapping so both top-level and delegated child runs emit `dogpile.run` spans automatically.
- `onRunComplete` fires only at root depth to avoid double-counting; sub-run costs use `onSubRunComplete` from the parent emit closure.
- Phase 6 is the only event-shape change in this milestone; all other phases are pure additions or engine-option injections.

**Issues Resolved:**
- Phase 6 `at` field removal: discriminator-safe fallback pattern established for RunEvent timestamp readers.
- Phase 9 span ancestry: `subRunSpansByChildId` WeakMap lookup via narrow coordinator callback shape change.
- Phase 10 partial abort metrics: aborted snapshots preserve observed partial counters; failed child costs excluded from parent own metrics.
- Phase 9 code review: `dogpile.agent-turn` spans use per-turn model-call accounting; failed `dogpile.run` spans retain best-effort attributes.

**Issues Deferred:**
- Per-turn health streaming (health emitted as events during a run)
- `compactProvenance` deduplication mode
- Built-in OTLP HTTP exporter
- `DogpileSpan.end()` timestamp parameter (OTEL span timing uses wall-clock, not P6 provenance timestamps)

**Technical Debt Incurred:**
- Phase 7 SUMMARY frontmatter missing `requirements-completed` field (documentation gap only; VERIFICATION.md is authoritative)
- 2 of 3 integration warnings resolved by post-audit commits (WARNING-1, WARNING-2); WARNING-3 (span timing) accepted as design tradeoff
