<objective>
Implement BUDGET-01 cancellation propagation. Replace the current direct
`signal: options.signal` passthrough at coordinator.ts:878 with a derived
per-child `AbortController` that forwards parent abort reasons. Lock the
`detail.reason` vocabulary on `code: "aborted"` errors (`"parent-aborted"`
half — `"timeout"` half lands in 02-02). Add observable marker for parent
aborts that occur AFTER a sub-run completes.

Purpose: Phase 2 success criterion 1 — parent cancel cascades to children;
child surfaces typed `code: "aborted"` with `detail.reason: "parent-aborted"`;
sub-run-failed event captures it; post-completion aborts are still observable
in replay (via streaming subscribers — see "D-10 marker observability" below).
</objective>

<d10_marker_observability>
**Locked decision (post-revision):** the marker is observable on
`Dogpile.stream()` subscribers, NOT on the rejected `run()` error.

Why: a non-streaming `run()` that throws does NOT preserve its parent events
array on the rejected error (verified — `src/runtime/engine.ts:230-239`
rejects with `runtimeError` only; the streaming-mode `emittedEvents` array is
local to the stream handle). Two paths considered:

  (a) Attach `partialTrace` to the rejected error similar to
      `sub-run-failed.partialTrace`. Out of scope — would require modifying
      `engine.ts`'s rejected-run shape, which is BUDGET-01-scope-creep and
      changes a separate public-surface invariant.

  (b) Emit the marker via `parentEmit(...)` BEFORE the throw. Streaming
      subscribers receive it; the rejected promise still throws the abort
      error. **Selected.**

Implication for Test B: drive the scenario through `Dogpile.stream()`,
collect events via a `subscribe` callback (NOT `await result`), assert the
`sub-run-parent-aborted` event arrives on the subscriber before the run's
final error event. Document inline in the test that non-streaming `run()`
sees only the thrown error, by design.
</d10_marker_observability>
