<!-- Phase 1 facts that Phase 2 BUDGET-04 simply locks: -->

1. `teedEmit` (coordinator.ts:846) pushes child events to a LOCAL `childEvents` buffer
   AND to `options.emit?.(event)` (the original streaming consumer). It does NOT
   push child events into the parent's events array used by termination evaluation.
   The parent's events array only receives the three parent-emitted variants:
   `sub-run-started`, `sub-run-completed`, `sub-run-failed`, plus the new
   `sub-run-budget-clamped` (02-02) and `sub-run-parent-aborted` (02-01) markers.

2. Phase 1 D-18: every successful sub-run pushes ONE synthetic transcript entry
   with `agentId: "sub-run:<childRunId>"` and `role: "delegate-result"`.
   `protocolProgress` (termination.ts:449-451) returns
   `protocolIteration ?? iteration ?? transcript.length`. Therefore one
   `sub-run-completed` counts as one parent iteration via the synthetic entry.

3. `protocolMinTurns` (termination.ts:453-464) reads protocol.minTurns from
   the protocol config at evaluation time. Per-instance config naturally means
   per-instance floors — the same protocol kind in parent vs child carries
   different floors because they are different config objects.

```ts
// Returns a ConfiguredModelProvider that produces protocol-aware text:
//   - protocol === "broadcast" → "<role>:<agentId> independently assessed the broadcast mission."

//   - protocol === "coordinator" + phase === "plan" (default) → "planned the coordinator-managed mission."
//   - sequential (default) → frame/refine based on prior contributions

export function createDeterministicSequentialTestMission(model?): DogpileOptions;
// (And broadcast/shared variants — see file for full list.)
```

**Implication for Task 1:** the deterministic provider does NOT support a
"return decision X for N turns then Y" script. To drive a child that emits
"50 agent-turn events", configure the CHILD's protocol with `maxTurns: 50`
(sequential) or `minTurns + maxTurns: 50` (coordinator). The provider returns
deterministic text for each turn; the protocol's own iteration limit drives
the count. To drive a parent that issues exactly one `delegate` decision
followed by `participate` decisions, the parent must use a COORDINATOR
protocol — coordinator's plan agent decides delegate vs participate based on
the prompt, and the deterministic provider's "plan" phase returns planning
text. **The test will need to either use a custom small in-test provider that
issues a delegate decision deterministically, OR rely on existing test helpers
in coordinator.test.ts that already drive delegate decisions** (read
coordinator.test.ts existing tests first — many already do this via
`createDeterministicModelProvider` plus a coordinator config that produces
delegate decisions in plan-phase output, OR via a small inline provider that
returns decision JSON in `text`). Pick whichever pattern matches the existing
file's convention.

    ```ts
      // Inline coordinator-decision provider — extends baseProvider's `generate`

      // (Pattern: read budget-first-stop.test.ts to confirm the codebase's
      // canonical "scripted decision" provider shape and reuse it. If no
      // canonical helper exists, define this small one inline here.)

      const provider = {
        id: "budget-isolation-provider",
        async generate(request) {
          const phase = request.metadata?.phase;

            parentPlanCalls += 1;
            if (parentPlanCalls === 1) {

                  protocol: "sequential",
                  intent: "Run 50 child turns to verify parent isolation."

        }

        agents: [
          { id: "coordinator", role: "coordinator" }

        // Child sequential protocol is configured via the delegate decision's
        // `protocol` field (sequential). To force the child to 50 agent-turns,
        // pass `decision.budget.maxIterations: 50` in the delegate JSON above

        // src/runtime/decisions.ts before finalizing the inline JSON.

      const childAgentTurns = childResult.trace.events.filter(
        (e) => e.type === "agent-turn"
      );
      expect(childAgentTurns.length).toBe(50);
    });
    ```
